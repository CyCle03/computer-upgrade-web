## 2. 재화 체계

원작에는 **세 종류의 재화 개념**이 있습니다.

```
미네랄(원)  ──[10,000,000 : 1]──▶  Normal 코인(가스 UI)  ──▶  일반 상점(부품 구매)
         │                              │
         │                         강화는 비용 없음(확률만)
         │                              │
         └─ UI: 1원 / 1만원(=10,000원) 표기 ─┘
                                   환생 시 초기화

SCA 코인  ──▶  SCA 센터(영구 성장)  ──▶  환생해도 유지 (SCA 런처 저장)
```

> **환율 주의:** ~~미네랄 10,000 = 1코인~~ 은 **오류**입니다. 유튜브 공략·가이드 UI 기준 **천만 원(10,000,000) 미네랄 = Normal 코인 1** (`MINERAL_PER_COIN = 10_000_000`).  
> `1만원`은 **미네랄 10,000**의 **표시 단위**일 뿐, 코인 1개와 같지 않습니다.

### 2.1 미네랄 (원)

- 사냥터·작업 등으로 획득하는 **기본 자원** (게임 내 숫자 = **원 1:1**)
- UI 표시: `123원`, `1.5만원`(= 15,000 미네랄) 등
- **1,000만** 미네랄이 쌓이면 **Normal 코인 1**로 **자동 환전** (v1.1.1: 「게임 중 미네랄이 코인으로 환전」)

### 2.2 Normal 코인 (인게임 코인 · 가스 UI)

| 항목 | 내용 |
|------|------|
| **표시** | 스타 UI **가스** 칸 |
| **환전** | **미네랄 10,000,000(천만 원) = Normal 코인 1** |
| **용도** | 일반 상점 **부품 구매** (표 `cost` = **C**, 코인 개수) |
| **강화** | **코인/미네랄 소모 없음** — 확률 판정만 (실패 시 파괴·하락) |
| **초기화** | **환생 시** 부품·다운로드·미네랄·Normal 코인 리셋 |

**코인 ↔ 원 환산 예:** CPU 1강 `1C` = 코인 1개 = 미네랄 **1,000만** 필요(환전 후 결제). 메인보드 `150C` = **15억** 미네랄 상당.

### 2.3 SCA 코인 (클라우드 영구 재화)

| 항목 | 내용 |
|------|------|
| **저장** | SCA 런처 서버 (웹 복원판: PostgreSQL `permanent_currencies`) |
| **획득** | 환생 보상, 파티 사냥터 단계 보상 등 |
| **용도** | SCA 센터 영구 성장 (§4 참고) |
| **초기화** | **환생해도 유지·누적** |

### 2.4 웹 복원판 재화 (Normal 코인 생략)

웹은 스타 가스 UI가 없으므로 **Normal 코인 중간 화폐를 쓰지 않습니다.**

| 항목 | 웹 정책 |
|------|---------|
| **표시·결제** | 가이드 센터 **원화 구매가** = `tier.cost` = **미네랄(원) 1:1** |
| **환전** | 자동 환전·Normal 코인 패널 **없음** |
| **SCA 코인** | SCA 센터·환생 보상 등 **원작과 동일** (영구 재화) |
| **표기** | `formatMineral`: 10,000 이상 → `N만원` |

## 4. SCA 센터 · SCA 코인 상점

### 4.1 환생 시작 미네랄 (확인된 동작)

SCA 상점에서 구매한 **미네랄 수치가 합산**되어, **다음 환생 시 그만큼 들고 시작**합니다. (랜덤 범위 아님)

| SCA 코인 | 효과 (1회 구매당) | 최대 구매 |
|----------|-------------------|-----------|
| **500** | 환생 시작 미네랄 **+500** | 1회 |
| **800** | 환생 시작 미네랄 **+200** | 5회 |
| **5,000** | 환생 시작 미네랄 **+2,000** | 3회 |
| **8,000** | 환생 시작 미네랄 **+7,500** | 2회 |

**합산 예시**

- +500 1회만 구매 → 다음 환생 **500원** 시작
- +500 1회 + +200 2회 → **900원** 시작
- 전부 최대 구매 → **22,500원** (v1.1.4 상한 30,000 → **v1.1.9~1.2.9 상한 1,000,000**)

웹 복원판 계산 (`calcRebirthStartMinerals`):

```
시작 미네랄 = min(1000000,
  rebirthMineral500 × 500
+ rebirthMineralMax200 × 200
+ rebirthMineralMax2000 × 2000
+ rebirthMineralMax7500 × 7500)
```

> **버전별 변천 (공식 패치노트)**  
> - **v1.0.3:** SCA 상점 「시작 미네랄 증가」— 환생 후 시작 미네랄 **+10** (최초 도입)  
> - **v1.0.7:** 명칭 변경 → 「환생 미네랄 증가」  
> - **v1.1.4:** 환생 미네랄 **최대 30,000**  
> - **v1.1.9:** 환생 미네랄 **최대 1,000,000**, 숫자 `[0]` 길게 눌러 **연속 구매**  
> - 현재 UI(SP 표기)의 +500/+200/+2000/+7500 티어는 중간 버전부터 누적된 구조

## 5. 환생 (Rebirth)

| 항목 | 내용 |
|------|------|
| **조건** | GPU **10강** 장착 |
| **초기화** | 부품, 다운로드, 미네랄, **Normal 코인** |
| **유지** | **SCA 코인**, SCA 상점 구매 효과, **환생 수치** |
| **시작 미네랄** | SCA 상점 구매분 **합산 고정** (§4.1) |
| **보상** | 성능·진행도에 비례 **SCA 코인** (정확 공식 미공개) |
| **버프** | **환생 수치**에 따라 자동 미네랄 수입 등 영구 배율 증가 |

### 5.1 환생 수치 (누적 포인트)

원작 UI 「환생수치」는 환생 **횟수**와 별개의 **누적 수치**입니다.  
외형/커스텀 해금 조건:

| 환생 수치 | 해금 |
|-----------|------|
| 1,000만+ | 발키리 외형 등 |
| 3,000만+ | 추가 커스텀 |
| 5,000만+ | 추가 커스텀 |
| 7,000만+ | 추가 커스텀 |

> **웹 복원판:** 환생 시 장착 부품 성능 합산으로 `rebirthStat` 누적. **환생 SCA 보상/환생수치 보정은 구간표(보정 전 기준)로 적용**하며, 수입 배율 공식은 여전히 근사치입니다 (§8.2).

### 6.4 커뮤니티 참고 — 13배속방 · 7→10 자동구매 (재미 통계)

> **주의:** 제작자 공식 수치 아님. Discord/방제 플레이어 측정치로, 웹 복원판 튜닝 참고용.

#### 배속 공식 (29−N)

SCA 「게임 배속 +1프레임」은 **실시간 N배가 아닙니다.** EUD `Wait(29−N)` 계열:

| 배속 N | Wait = 29−N | 배율 `(29−3)/(29−N)` |
|--------|-------------|----------------------|
| 3 (기본) | 26 | **1.000×** |
| 7 | 22 | 1.182× |
| 10 | 19 | 1.368× |
| 13 | 16 | 1.625× |
| **15 (상한)** | **14** | **≈1.857×** (`26/14`) |

> v1.1.9 패치노트에는 SCA 상점 배속 **최대 13**까지 기록돼 있으나, 현재 플레이 기준 **SCA +12회 = 15**가 실질 상한이며 **15배속 = (29−3)/(29−15)** 로 계산합니다.

웹 `calcGameSpeedMultiplier`: **`(29−3) / (29−N)`** — ~~`N/3`~~ 사용 금지.

#### 13배속방 · 7강 자동구매 → 10강 (GPU 등급별)

| GPU 등급 | 7→10 기댓값 | ≈현실 시간 | 상위 1% | ≈시간 | 상위 99% | ≈시간 |
|----------|-------------|-----------|---------|-------|----------|-------|
| 엔트리 | 40,000 | ~5분 | 4,000 | ~30초 | 1,391,250 | 2시간+ |
| 메인스트림 | 160,000 | ~5분 | 16,000 | ~30초 | 5,565,000 | 2시간+ |
| 퍼포먼스 | 400,000 | ~5분 | 40,000 | ~30초 | 13,912,500 | 2시간+ |
| 하이엔드 | 1,000,000 | ~5분 | 100,000 | ~30초 | 34,781,250 | 2시간+ |

등급별 **기댓값 숫자**는 ×4 / ×10 / ×25 (엔트리 대비). **현실 시간(기댓값)** 은 모두 ~5분으로 통계 작성자가 역산.

**SCA 강화확률 마스터(+1%)** 기준: 기댓값 462,950 · ~2분19초 / 1% 4,650 · ~1.4초 / 99% 2,131,950 · ~10분+

**앵커:** 13배속방 · 자동구매 · 현실 **3초에 7강 200개** (구매/처리 속도 참고).

### 6.5 웹 수입 밸런스 (작업·사냥)

원작은 **타격(킬)마다** `mineralPerUnit × 유닛 수` 지급. 공격 주기 ≈ **GPU등급 프레임 ÷ 24fps**, SCA 배속 `(29−3)/(29−N)` 반영.

| 항목 | 원작 체감 | 웹 구현 |
|------|-----------|---------|
| 수입 트리거 | 킬/타격 1회 | `calcIncomeEventIntervalMs` ≈ **833ms** (엔트리 20f·배속3) |
| ~~고정 1초 틱~~ | — | 폐기 (너무 느림/빠름 불일치) |
| 작업+사냥 | 동시 | 동일 간격·별도 `calcWork/HuntIncomePerTick` |
| `mineralPerUnit` | 가이드·영상 값 | `WORK_TASKS` / `GAME_HUNTING` (§3.9) |

**초반 밸런스 검증 (배속3·엔트리·1작업+1사냥 유닛):**

- 수입 ≈ **(1+1) × 1.2회/초 ≈ 2.4원/초**
- 공랭 2강 **500원** → **~3.5분** (영상 「500원 쿨러」 구간과 유사)
- 심시티 **100원×4유닛×1.2 ≈ 480원/초** → 500원 쿨러 **~1초** (다단계 유닛 투입 시)

> RAM·CPU 늘리면 작업+사냥 유닛이 동시에 늘어 수입이 가속 — 원작과 동일. 고티어 사냥 `mineralPerUnit`을 **낮추지 말 것** (적 처치당 고액이 핵심).
