## 4. SCA 센터 · SCA 코인 상점

### 4.1 환생 시작 미네랄 (확인된 동작)

SCA 상점에서 구매한 **미네랄 수치가 합산**되어, **다음 환생 시 그만큼 들고 시작**합니다. (랜덤 범위 아님)

| SCA 코인 | 효과 (1회 구매당) | 최대 구매 |
|----------|-------------------|-----------|
| **500** | 환생 시작 미네랄 **+500** | 1회 |
| **800** | 환생 시작 미네랄 **+200** | 5회 |
| **5,000** | 환생 시작 미네랄 **+2,000** | 3회 |
| **8,000** | 환생 시작 미네랄 **+7,500** | 2회 |

**합산 예시**

- +500 1회만 구매 → 다음 환생 **500원** 시작
- +500 1회 + +200 2회 → **900원** 시작
- 전부 최대 구매 → **22,500원** (v1.1.4 상한 30,000 → **v1.1.9~1.2.9 상한 1,000,000**)

웹 복원판 계산 (`calcRebirthStartMinerals`):

```
시작 미네랄 = min(1000000,
  rebirthMineral500 × 500
+ rebirthMineralMax200 × 200
+ rebirthMineralMax2000 × 2000
+ rebirthMineralMax7500 × 7500)
```

> **버전별 변천 (공식 패치노트)**  
> - **v1.0.3:** SCA 상점 「시작 미네랄 증가」— 환생 후 시작 미네랄 **+10** (최초 도입)  
> - **v1.0.7:** 명칭 변경 → 「환생 미네랄 증가」  
> - **v1.1.4:** 환생 미네랄 **최대 30,000**  
> - **v1.1.9:** 환생 미네랄 **최대 1,000,000**, 숫자 `[0]` 길게 눌러 **연속 구매**  
> - 현재 UI(SP 표기)의 +500/+200/+2000/+7500 티어는 중간 버전부터 누적된 구조

| 항목 | 내용 |
|------|------|
| **조건** | GPU **10강** 장착 |
| **초기화** | 부품, 다운로드, 미네랄, **Normal 코인** |

외형/커스텀 해금 조건:

| 환생 수치 | 해금 |
|-----------|------|
| 1,000만+ | 발키리 외형 등 |
| 3,000만+ | 추가 커스텀 |
| 5,000만+ | 추가 커스텀 |
| 7,000만+ | 추가 커스텀 |

> **웹 복원판:** 환생 시 장착 부품 성능 합산으로 `rebirthStat` 누적. **환생 SCA 보상/환생수치 보정은 구간표(보정 전 기준)로 적용**하며, 수입 배율 공식은 여전히 근사치입니다 (§8.2).

## 6. 사냥터 · 수입

### 6.1 작업 사냥터 vs 파티 사냥터

| 구분 | 수입 | 비고 |
|------|------|------|
| **작업** | 미네랄(원) 자동 | 게임 다운로드로 상위 단계 해금 |
| **파티** | 미네랄 + **SCA 코인** | 파티 ON 시 작업 수입 중단 (상호 배타) |

원작에는 **수동 일꾼 채굴**(SCV/Probe/Drone) UI가 없고, 작업·파티 **자동 수입**이 핵심입니다.

### 6.2 파티 사냥터 티어 (원작 UI 수치)

**1번 라인 — 원:** 30 → 300 → 3,500 → 50,000 → 750,000 → 1,500,000  
**1번 라인 — SCA:** 2 → 20 → 200 → 2,000 → 5,000 → 40,000  

**2번 라인 — 원:** 1 → 100 → 2,500 → (30,000 → 250,000 → 5,000,000)  
**2번 라인 — SCA:** 20 → 500 → 7,500  

웹 복원판: `PARTY_HUNTING_TIERS` 9단계 (1-1 ~ 2-3) 구현.

### 6.3 오버클럭 사냥터

- DDR4/DDR5 오버클럭 파츠, 연구소 레벨, 자동 사용 ON/OFF

| 항목 | 웹 근사 로직 |
